O projeto Jogos de \doxylink{class_tabuleiro}{Tabuleiro} é uma aplicação desenvolvida como trabalho final da disciplina de Programação e Desenvolvimento de Software II (PDS II) do curso de Sistemas de Informação na UFMG. Este projeto implementa um sistema para jogar diferentes jogos, proporcionando funcionalidades cadastro de jogadores, exclusão de jogadores, estatísica dos jogadores e a listagem dos jogadores. Os jogos disponibilizados para serem jogados são\+:
\begin{DoxyItemize}
\item Lig4
\item \doxylink{class_reversi}{Reversi}
\item Tic Tac Toe (jogo extra)
\item Jogo da Memória (jogo extra)
\item Campo Minado (jogo extra)
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysection{\texorpdfstring{Equipe}{Equipe}}\label{md__r_e_a_d_m_e_autotoc_md2}

Durante o desenvolvimento deste projeto, a equipe enfrentou alguns desafios\+:


\begin{DoxyItemize}
\item Uso do Git e Github\+: A utilização dessas ferramentas apresentou-\/se como um desafio para a equipe. No decorrer do projeto, tivemos dificuldades ao lidar com conflitos de merge, em entender o fluxo de trabalho nas branches e na utilização das funcionalidades para manter o código organizado.
\item Uso de Makefile\+: O processo de compilação dos arquivos com o Makefile apresentou-\/se com um desafio na reta final. Algumas das dificuldades enfrentadas foram\+: erro com o arquivo executável, conjutamente com um erro relacionado ao tipo binário gerado na compilação (Exemplo\+: Relocation against symbol \+\_\+zvcampominado can not be used when making a PIE object). Para lidar com o problema utilizamos o comando {\ttfamily make clean} e depois realizamos o processo de compilação novamente com {\ttfamily make}. Além disso acrescentamos no arquivo Makefile a linha\+: {\ttfamily CFLAGS=-\/std=c++11 -\/Wall -\/f\+PIC}, caso o problema estivesse relacionado a um trecho que envolva código compartilhado ou parte de uma biblioteca.
\item Realização dos Testes\+: Compreender e executar os testes também se mostrou um processo desafiador. Houve dificuldades em entender a estrutura e como conseguir integrá-\/los no nosso processo de desenvolvimento.
\item Salvar informações no arquivo Jogadores.\+txt\+: No processo de manipulação do arquivo, o grupo apresentou dificuldades para tratar um erro específico ao excluir um jogagor, o que gerava uma linha em branco no final do arquivo,. A presença desta linha acarretava o não funcionamento do executável, exibindo o seguinte errro\+: `terminate called after throwing an instance of \textquotesingle{}std\+::invalid\+\_\+argument' what()\+: stoi Aborted.\`{} Para o bom funcionamento do código, é necessário retirar as linhas vazias do arquivo. O grupo buscou tratar essa dificuldade por meio de uma função que posiciona o cursor no início do arquivo, para assim evitar que ele chegue ao final.
\end{DoxyItemize}
